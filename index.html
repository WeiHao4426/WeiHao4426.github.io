指针的用法：
声明一个指针变量：int tpr;
声明一个变量A：int A；
将指针指向A的地址：tpr = &A；
输出A的地址：printf（”%p“，tpr）；
输出A储存的值：print发（”%d“，tpr）；

指针与数组的关系：
int b[5];
int *btpr;

btpr = b;等效于btpr = &b[0];

b[3] = *(b + 3) = btpr[3] = *(btpr +3)

指针的运算应该在连续内存区（数组）中进行；
算数、比较、赋值运算；
指针运算偏移量：
例如：int tpr，tpr+1 后偏移量为sizeof（int）；

标志物设计：要在大范围内可见，并且给予玩家一定的引导。
引导线：可以在画面中用地形或物体边线、管道、线缆之类来引导玩家的前进方向，适合重新引导方向

美术表现设计：
光线的从处理：玩家会对块状光源有探索欲望，用于引导。
场景的布置：过多的重复会让玩家下意识的注意该方向，有可能误导；地上的打光会让玩家注意到这个路。
动态、光效、细节：
用破碎的岩石壁提醒玩家“此处有东西”；
关于颜色和形状的设计，如绿色代表安全，……

感官规则：感官规则最好符合玩家常识和游戏背景；要尽量给予正面反馈。但是它只能作为辅助信息，不能作为主线。

人类一败涂地

发表于 2021-11-02 更新于 2023-06-11

tags: [游戏策划]
categories: [游戏策划]

正好最近万圣节促销，买了人类一败涂地这款游戏，正在熟悉操作并过关，那就结合我游玩的过程说一下它的关卡设计吧。

首先，这款游戏的操作教学都是融入到前几个关卡的。新玩家在初次进入游戏时，直接控制一个小白人，然后依靠自己的经验去进行摁下开关、搬动箱子到开关上、拉下扳手等等操作。这些都是游戏的基本操作，玩家是完全可以根据自己的日常生活经验来推断出来的。

然而，完成这些基础的教学关卡后，玩家会遭遇第一个难题：火车关（至少对于我是难题）。玩家需要爬上一个火车，然后一步步通过火车登上平台。对我来说，这一关有点复杂了。我甚至不清楚“将视角向下拉可以爬上平台”这样的基础操作（前面的关卡没教），这个关卡确实是一个难度的提升。当我查询百度上的教程，并最终通过捷径（翻墙）到达下一关时，我才获得了相关操作的知识。

第一关：豪宅

第一关豪宅是新手引导关，关卡难度并不难。
在第一关中，玩家主要需要了解移动视角、按下按钮、以及使用黄色长方体来获取教学的各种简单操作。
关卡流程：

  1. 进入到豪宅关卡的场景，地面上有一个遥控器。拿起遥控器后，场景内会出现一个幕布，上面会出现教程动画
  2. 顺着路来到一扇门前，这是一扇常见的按键门，将视野对准门边的按键按下抓取键即可开门。
  3. 进入按键门顺着小道走，捡起这里的遥控器，出现教程动画：如果想要拿取较高处或者低处的物品，需要调整视野中心位置的高低。
  4. 来到如上开关门前，此处有上下两个开关，调整视角分别按下两个开关即可开门。
  5. 继续前进来到关卡出口处，此时按键门的按键有两个，水平分布。站在开关前同时按下左右手抓取按键即可同时按下开关，出口处的门会被打开，走近门从出口跃下即可结束本关卡

第二关：火车以及第三关：搬运第四关：山峰

对我这个新手来说，这几关的难度明显提升很多，我花了好多时间才通过。
在这几关中，玩家需要自己领悟：

  • 移开箱子发现门
  • 爬到火车上并前进到平台上（如果没有找到门）
  • 站在箱子上，抓住柱子并用脚移动车厢
  • 以及其他基本的结合性操作
  • 举起双手冲刺以爬上更高的平台
  • 甚至实在过不去时，在网上搜索翻墙的方法（我属于这种）

这一关除了让玩家把之前的基础操作结合起来，更重要的是让玩家熟悉这个游戏的谜题设计套路。这个套路是贯穿这个游戏所有关卡的。它包括：

把门隐藏到某个东西后面
推某个有轮子的东西，产生变化
某个开关需要重物在上面压着
按下按钮
站在跷跷板上，用自己的质量使跷跷板运动起来
（大概这么多了）

所以，第一关是让玩家熟悉基本的操作，那么第二关就是让玩家体会游戏中谜题的设计，熟悉游戏谜题设计者的思路，达到上手游戏的效果。

第五关：拆除

在前几关，玩家基本懂了游戏的基本操作和迷题的设计思路。此时，玩家将面对几个连续的解密小关卡。它们组合在一起，就形成了第一个正式的关卡。

这个关卡包括以下部分：

  1. 第一个场景内有一个被木条封住的门，抓住木条向后拽即可拆除木条，拆掉后进入门。
  2. 第二个场景，房间内有一扇落地窗，先将落地窗前的摇杆拉至最右侧，窗外吊着的重物会向右侧移动利用自重砸开墙壁。双手将墙角的灭火器举高高冲向落地窗，即可将落地窗砸碎来到屋外。
  3. 通过碎裂的墙壁进入到下一个场景，跃起抱住铁球利用惯性适时松开抓取键即可荡到对面。
  4. 【然后我就卡在这里了......等我打过这一关再说吧。

以上，游戏”人类一败涂地”的新手关卡及第四关梳理。

做减法：游戏设计要以简洁为好，不要加入过多元素，把选择权交给玩家。

只设计一个核心玩法，然后加入几个必要元素即可。此外，直观性也十分重要。最好让玩家可以直接看到游戏的目标和博弈，而不是需要介绍一堆新手难懂的规则。

1：背景
宇宙由帝国统治了数百万年，在人工智能的帮助下，人类依靠空间折跃站技术占领了银河系。不过，当寿命与帝国一样长的皇帝雷托一世突然死亡后，整个银河陷入了混乱：人工智械纷纷关闭，依托它们的空间折跃站也都停转，一个个星球彼此间断开了联系……
756个银河标准年后，一个偏远的星系中的秘密克隆仓库内，全银河唯一一台未关闭的人工智械“伊甸”，唤醒了一个克隆体-爱达荷，并且告诉他，他的目的就是要回到帝国首都，找出一切混乱的根源。

2：设定：
爱达荷：主角，记忆为空白，但始终坚持寻找大混乱的起因，并对帝国十分忠诚
伊甸：毒舌机器人，在游戏中类似向导作用，玩家也可以通过与它对话获取背景知识。
雷托一世：在人族内战时期依靠杰出的军事才能，从银河系外围一直攻到银河系中心星球（即现在的银河帝国首都“终点星”），并建立了银河帝国，依靠机械辅助技术统治了数百万年。
游戏中遇到的敌人：
拾荒者：大混乱后受到核污染而出现的短命人类，极端好战。
海盗：不用多说的吧
教廷军队：狂热地拥护教皇，认为帝国军队是造成大混乱的起因，故而仇视主角。

3.游戏剧情
爱达荷从银河一条旋臂末端的克隆站中诞生，驾驶飞船一路向银河中心进发。一路上会遇到各种敌人及突发事件，并且会在事件结束后跳出背景故事。
随着主角渐渐接近银河帝国首都，接收的背景知识越来越与皇帝雷托一世相关。
当玩家修好最后一个空间折跃站，进入帝国首都“终点星”时，玩家会收到一段来自皇帝本人的遗留视频，具体内容：
银河中每天都有兆亿的事件发生……每天都有无数抉择要去判断。即使有智械的辅助，我的记忆体最终还是故障了。原来，那些太过久远的记忆，连智械都无法清除。
所以，我意识到，是时候换个人了。
我在银河系每条旋臂的末端都放置了一个克隆仓库，并秘密下令军方关闭所有智能机械……除了爱达荷们的“伊甸”。
为了数百万年的长治久安，七百多年的混乱不值一提。
当一个爱达荷最先到达终点星时，他的实力必然已经达到了我在人类时期的水平。其他爱达荷们会被销毁，而那个最先到达的，则会成为帝国的下一任皇帝……
所以恭喜你，爱达荷5号。你现在可以称自己为：雷托二世。
祝愿你的统治经久不衰。
4.游戏玩法

左下为玩家出生点。四角星为星系，只能从一个星系通过折跃站跳到相邻星系。五角星为终点星。
游戏玩法可以类似于“哀恸之日”的战棋类

  某些星系中有维修站，玩家可以在里面购买战斗机或是进行维修。
  在战斗中，通过用战斗机在棋盘上移动到敌舰旁边，攻击敌舰来获得胜利。

读书笔记

==如何阅读一本书==

第一章 阅读的活力与艺术

  • 阅读的目标：为获得资讯而读，以及为求得理解而读
  • 第一，阅读可以是一件多少主动的事。第二，阅读越主动，效果越好。
  • 阅读的目标：为获得资讯而读，以及为求得理解而读
    • 只有一种方式是真正地在阅读。没有任何外力的帮助，你就是要读这本书。你什么都没有，只凭着内心的力量，玩味着眼前的字句，慢慢地提升自己，从只有模糊的概念到更清楚地理解为止。这样的一种提升，是在阅读时的一种脑力活动，也是更高的阅读技巧。这种阅读就是让一本书向你既有的理解力做挑战。
  • 我们可以将“阅读”这个词，区分成两种不同的意义。（在读者与作者的理解力差距方面）（吸收资讯或提升理解力） 　　第一种意义是我们自己在阅读报纸、杂志，或其他的东西时，凭我们的阅读技巧与聪明才智，一下子便能融会贯通了。这样的读物能增加我们‘的资讯，却不能增进我们的理解力，因为在开始阅读之前，我们的理解力就已经与他们完全相当了。否则，我们一路读下来早就应该被困住或吓住了—这是说如果我们够诚实、够敏感的话。 　　第二种意义是一个人试着读某样他一开始并不怎么了解的东西。这个东西的水平就是比阅读的人高上一截。这个作者想要表达的东西，能增进阅读者的理解力。这种双方水准不齐之下的沟通，肯定是会发生的，否则，无论是透过演讲或书本，谁都永远不可能从别人身上学习到东西了。这里的“学习”指的是理解更多的事情，而不是记住更多的资讯—和你已经知道的资讯在同一水平的资讯。
  • 阅读就是学习：指导型的学习，以及自我发现型的学习之间的差异

第二章 阅读的层次

  • 一般来说，阅读的规则是：努力越多，效果越好。至少在阅读某些超越我们能力的书时，花一点力气就能让我们从不太了解进升到多一些了解的状态。
  • 一共有四种层次的阅读（阅读的层次是渐进的。）
    • 第一层次的阅读，为基础阅读。一个人只要熟练这个层次的阅读，就摆脱了文盲的状态，至少已经开始认字了。在熟练这个层次的过程中，一个人可以学习到阅读的基本艺术，接受基础的阅读训练，获得初步的阅读技巧。在这个层次的阅读中，要问读者的问题是：“这个句子在说什么?”
    • 第二个层次的阅读，为检视阅读，特点在强调时间。在这个阅读层次，学生必须在规定的时间内完成一项阅读的功课。譬如他可能要用十五分钟读完一本书，或是同样时间内念完两倍厚的书。即在一定的时间之内，抓出一本书的重点—通常是很短，而且总是过短，很难掌握一本书所有重点。这个层次要问的典型问题就是：“这本书在谈什么?”
    • 第三种层次的阅读，为分析阅读。比起前面所说的两种阅读，这要更复杂，更系统化。随内文难读的程度有所不同，在使用这种阅读法的时候，多少会相当吃力。分析阅读就是全盘的阅读、完整的阅读，或是说优质的阅读—你能做到的最好的阅读方式。如果说检视阅读是在有限的时间内，最好也最完整的阅读，那么分析阅读就是在无限的时间里，最好也最完整的阅读。
    • 第四种，为主题阅读。这是所有阅读中最复杂也最系统化的阅读。这种阅读要求非常多，就算阅读的是一本很简单、很容易懂的书也一样。在做主题阅读时，阅读者会读很多书，而不是一本书，并列举出这些书之间相关之处，提出一个所有的书都谈到的主题。但只是书本字里行间的比较还不够。主题阅读涉及的远不止此。借助他所阅读的书籍，主题阅读者要能够架构出一个可能在哪一本书里都没提过的主题分析。

第三章 阅读的第一个层次：基础阅读

  • 学习阅读的阶段:儿童具备纯熟的阅读能力之前，至少会经历大约四个截然不同的阶段。
    • 第一个阶段被称为“阅读准备阶段”,包括了几种不同的学习阅读的准备工作。身体方面的准备，包括良好的视力与听力。智力方面的准备是要有起码的认知能力，以便孩子能吸收与记住一个字，与组成这个字的字母。语言上的准备包括口齿清晰，能说出一些正确的句子。个人的准备，则包括能与其他孩童一起学习的能力，保持注意力，服从等等。
      • 第二个阶段，孩子会学习读一些简单的读物。
    • 第三个阶段:快速建立字汇的能力，所用的方法是从上下文所提供的线索，来猜测不熟悉的字眼。
    • 第四个阶段的特征是精练与增进前面所学的技巧。最重要的是，学生开始能消化他的阅读经验—从一本书所提出来的一个观点转化到另一个观点，在同一个主题上，对不同的作者所提出来的观点作比较。
    （这里所列举的四个阶段，都属于我们在前一章所谈的、第一个层次的阅读。这些阶段，都是基础阅读）

第四章 阅读的第二个层次：检视阅读

  • 注：基础阅读是包含在检视阅读中的，而事实上，检视阅读又包含在分析阅读中，分析阅读则包含在主题阅读中。
  • 检视阅读一：有系统的略读或粗读
    • 先看书名页
    • 研究目录页
    • 如果书中附有索引，也要检阅一下
    • 读一下出版者的介绍
    • 从你对一本书的目录很概略，甚至有点模糊的印象当中，开始挑几个看来跟主题息息相关的篇章来看。
    • 最后一步，把书打开来，东翻翻西翻翻，念个一两段.有时候连续读几页，但不要太多。
      • （一种格式化的概览阅读法）
  • 检视阅读二：粗浅的阅读
    • 头一次面对一本难读的书的时候，从头到尾先读完一遍，碰到不懂的地方不要停下来查询或思索。只注意你能理解的部分，不要为一些没法立即了解的东西而停顿。继续读下去，略过那些不懂的部分，很快你会读到你看得懂的地方。集中精神在这个部分。
    （对之后第二次阅读有帮助）
  • 阅读的速度:所谓阅读速度，理想上来说，不只是要能读得快，还要能用不同的速度来阅读—要知道什么时候用什么样的速度是恰当的

==游戏设计之道==

第一关

  1. （电子)游戏的定义：
    • 至少需要一名参与者
    • 有一定规则
    • 有胜利条件
  2. 电子游戏简史
  3. 游戏分类
    • 动作类
    • 射击类
    • 冒险类
    • 建筑/经营
    • 模拟养成
    • 策略
    • 音游
    • 机车模拟
    • 益智
    • 体育
    • 其他
  4. 一个游戏团队的成员
    • 程序员
    • 美工
    • 设计师
    • 制作人
    • 测试
    • 质量保证
    • 作曲/音效
    • 剧作者
    • 产品经理
    • 创意经理
    • 技术总监
    • 其他

第二关 创意

  • 获得及把握创意
    • 好的创意与愚蠢的创意只有一线之隔
    • 读平时不会读的东西，拓宽知识面，接触新事物
    • 讲了一些用来放松自己，获得灵感的方法
    • 听演讲
    • 玩个烂游戏，思考如何将它变好，从中吸取教训
    • 追随自己的热爱
  • 搞清楚：玩家想要什么
  • 关于趣味性
    • 无趣理论说的是，先想出一个“有趣”的创意，然后在开发这个创意的过程中，如果你发现某个东西不太有趣（甚至感觉是无趣的），那么就应该立刻把它从游戏里去掉。当你把所有这些有趣的内容都去掉后，剩下的部分就都是有趣的了。
    • 学会用挑剔的眼光去看待这些创意
  • 头脑风暴
    • 为了构思创意，头脑风暴是不可或缺的。有效的头脑风暴需要做到以下五点： 1.头脑要能正常转。 2.有能写的东西。 3.有可以随意写写画画的东西。 4.一个能办事的环境。 5.和你一样善于开动脑筋的同伴。
    • 在头脑风暴之前一定要明确几点： 1.不要先入为主，没有不好的或者愚蠢的创意，头脑风暴需要肯定所有创意。 2.参与者所涉及的领域尽可能多，不仅是游戏设计师，最好把程序、美工、测试、运营、音效……不管是谁都尽量参与进来。 3.考虑所有想在游戏里添加的元素，并将它们记录下来。要明确头脑风暴的目标是获得尽可能多的创意，如果偏离了主题大可扶正一下，但是不要矫枉过正。
  • 如何重新获得灵感
    • 完善时间规划
    • 当需要设计一个不太令你兴奋的功能时应该考虑将它放在后面再做，直接跳到那些你感兴趣的部分。
    • 换一个环境放松一下自己

第三关 为游戏编故事

  • 故事性和游戏性，哪个更重要？
    • 游戏不是必须要有故事，但是游戏一般会讲述一个故事
  • 一个基本的故事框架
    • 主角想完成某件事
    • 主角遇到难题
    • 主角试图克服难题
    • 更大的难题，更大的危险
    • 最终的难题（BOSS）
    • 最终解决难题
  • 要设计出多条让玩家乐在其中的流程
  • 剧情三角论
    • 角色（绿野仙踪）
    • 事件（巨蟒与圣杯）
    • 背景（星球大战）
    • 三者只能突出一个（大多数情况）
  • 让故事服从游戏性
  • 在你觉得玩家心满意足的时候结束游戏。你可以弄些隐藏剧情来提高游戏的重复可玩性，或者埋些伏笔，给续作开个头。但最后一定要对玩家厚道一些，让他们觉得自己在游戏里完成了所有自己需要做的事。有些游戏也有一些额外模式，供玩家在游戏通关后还能继续玩。多重结局、小游戏、能解锁的内容或者可以下载的“被删掉的关卡”等，这些小玩意能够让玩家重新回到游戏世界，同时也不用再看一遍游戏剧情。其实，如果你能构建一个让玩家沉醉其中的世界，那么他们一定会反复玩。
  • 游戏时长，早期游戏平均20个小时通关。更早的时候一般是40个小时。现在大多数游戏是8到10小时，不过这个也根据不同游戏而有所不同
  • 游戏名最好简单好记
  • 角色的设计最好给玩家带来冲击感，并且拥有合适的名字。时刻想着角色的动机。

第四章 写文档

  1. 游戏设计文档（game design document），或者简写为GDD，是制作游戏的菜谱，不仅需要描述游戏是什么，还要包括游戏的做法。
    1. GDD基本分为四类：单页说明书、十页说明书、游戏流程表和游戏设计文档。
    2. 以十页说明书为例，里面包括游戏名称、游戏总览、角色、游戏可玩性、游戏世界、游戏体验、游戏机关、敌人、剧情过场、奖励内容。
  2. （？？？作者的吐槽）虽然写GDD需要耗费大量时间和精力，但没有人会想看它，甚至团队组员也不想读。为嘛？因为GDD又臭又长，里面的信息质量也良莠不齐——有的非常实用，有的形同“天书”。我第一次写GDD时就发现了，每个人都很关注它，但没人会当真愿意花大把时间把它看完。
  3. 多用游戏流程表这样的工具来协助你尽快找出设计上的问题。
  4. 根据游戏的预期ESRB定级进行设计。
  5. 有游戏关卡设计流程表可供参考，用来完善GDD。
  6. 游戏设计师最重要的工作就是交流！

总结：《游戏设计之道》是一个资深游戏人的经验之谈，教读者怎么实际去操作而不是讲理论上的东西，可以从中了解一个游戏如何从头到尾被制造出来。被称为游戏业者的工具书。
